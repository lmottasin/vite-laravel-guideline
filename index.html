What is Vite?

The next-generation frontend tool.

Vite is a fast build tool that replaces older tools like Webpack. It serves your files instantly during development and bundles your Javascript and CSS into highly optimized files for production.

The Laravel Plugin

Bridging the gap between PHP and JS.

Vite by itself doesn't know about Laravel. The plugin allows you to use the @vite() directive in Blade and handles the auto-refresh (HMR) magic.

vite.config.js 
import laravel from 'laravel-vite-plugin';
import react from '@vitejs/plugin-react';

export default defineConfig({
    plugins: [
        laravel({
            input: [
                "resources/js/app.js", 
                "resources/css/app.css"
            ],
            refresh: true,
        }),
        react(),
    ],
});

Critical Configuration Rule

If you add a new entry file (like admin.tsx), you MUST add it to the input array above. Otherwise, it will not be built.

Why pass TSX Files?

The most common mistake developers make.

When you have a modular app (e.g., Inventory, Admin, Shop packages), each has its own "Main" file. Vite is not a mind reader; it only looks at what you list in the input array.

Proper Multi-Module Config 
input: [
    // Global
    "resources/js/app.js",
    
    // Modules (CRITICAL)
    "packages/inventory/resources/assets/js/index.tsx",
    "packages/admin/resources/assets/js/index.tsx",
    "packages/shop/resources/assets/js/index.tsx"
]
Analogy: Think of the Input Array as the Guest List for a party. If your file isn't on the list, the bouncer (Vite Build) won't let it in, even if it's dressed up nicely.

Development Mode

The "Forgiving" Mode

  • Vite runs a live server.
  • Files are compiled on demand when the browser asks for them.
  • If you forget to list a file in config, Vite might still find it via imports.
npm run build

Production Mode

The "Strict" Mode

  • No server. Just static files.
  • Vite ONLY looks at the input config list.
  • Forgot a file? It doesn't get built. 404 Error.

Final Checklist for Entry-Level Devs

1

Vite = Speed

It serves instantly in dev and bundles compactly for prod.

2

Bundling is Merging

Don't ship 100 tiny JS files. Ship one big optimized one.

3

Don't Trust Localhost

Just because it works with npm run dev doesn't mean your config is correct.

!

The Golden Rule

If you want a TSX/JS file to be an entry point in Laravel, you MUST add it to the input array in vite.config.js.
